实现以下功能：
1、需求要求描述：
A、完成用户的注册，需要有用户姓名、身份证号、密码、性别、年龄、登录密码等信息。
业务规则：身份证号不允许重复；登录密码必须是6-20位的字母及数字混合，同时密码的存储要保证安全性（非明文）。
B、完成用户的登录操作，通过身份证号及密码进行登录系统；不合法用户提示错误信息。
C、登录后用户完成个人社保信息的录入登记，个人社保卡信息包括社保卡号、金融卡号；参保信息包括参保年份、参保金额、保险类型（农村医疗、职工医疗）
业务规则：社保卡号、金融卡号长数为20位的数字；参保信息按年进行参保记录。
D、登录后用户可以根据年份查询个人参保数据信息进行展现。
2、实现要求描述：
A、先按需求进行表模型设计，可使用对应的UML工具，如viso、pd等；
B、前端界面的要求可使用框架式，如layui、bootstrap等;
